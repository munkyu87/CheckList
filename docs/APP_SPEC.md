# 체크리스트 앱 스펙 (로컬 전용, React Native)

## 방향 요약
- **프론트엔드만**, 로컬 저장소(MMKV)만 사용
- **범용**: 경로당뿐 아니라 매장 점검, 현장 점검 등 어떤 용도로든 재사용
- **체크리스트 그룹**으로 템플릿 관리 → **방문/기록** 생성 시 날짜·**체크 대상명**·그룹 선택 → 항목 체크 + 비고  
  (대상명 라벨은 그룹별로 "경로당명", "매장명" 등 **커스텀** 가능)

---

## 1. 데이터 구조

### 1.1 체크리스트 그룹 (ChecklistGroup)
| 필드 | 설명 |
|------|------|
| id | 고유 ID (UUID 등) |
| name | 그룹 이름 (예: "경로당 방문 체크리스트") |
| subjectLabel | **체크 대상명** 라벨 (UI에 표시할 이름. 예: "경로당명", "매장명", "대상명". 비우면 기본값 "대상명") |
| createdAt | 생성일시 |

### 1.2 체크리스트 항목 – 그룹 소속 템플릿 (ChecklistItemTemplate)
| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| groupId | 소속 그룹 ID |
| title | 항목 제목 (예: "위생 상태 확인", "식사 제공 여부") |
| order | 그룹 내 표시 순서 |

### 1.3 방문/기록 (ChecklistRecord)
한 번의 방문(또는 한 번의 점검 세션)을 의미.

| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| date | 방문/점검 날짜 |
| subjectName | **체크 대상명** (경로당명, 매장명, 현장명 등 사용처에 맞게 입력. 화면 라벨은 그룹의 subjectLabel 사용) |
| groupId | 사용한 체크리스트 그룹 ID |
| createdAt | 기록 생성일시 |
| overallNote | 전체 비고 (선택) |

### 1.4 기록별 체크 항목 (RecordItem)
그룹의 각 템플릿에 대응하는 “실제 체크 + 비고”.

| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| recordId | 소속 기록 ID |
| templateItemId | ChecklistItemTemplate ID (어떤 항목인지) |
| checked | 체크 여부 (boolean) |
| note | 해당 항목 비고 (선택) |

## 2. 화면/기능 구성 제안

### 2.1 탭 또는 메뉴
1. **기록 (홈)** – 과거 기록 리스트 (날짜, 체크 대상명, 그룹명) → 탭하면 상세
2. **새 기록** – 날짜·체크 대상명·그룹 선택 후 체크리스트 진행
3. **체크리스트 그룹 관리** – 그룹 목록 → 그룹별 항목(템플릿) 관리

### 2.2 화면별 상세

#### 홈 – 기록
- 리스트: 날짜 | 체크 대상명 | 그룹명 (최신순 또는 날짜순)
- 항목 탭 → **기록 상세**: 해당 기록의 체크 항목 + 비고 보기 (수정 가능하면 좋음)
- 빈 상태 문구: "아직 기록이 없어요. 새 기록을 만들어 보세요."

#### 새 기록
1. **입력**: 날짜(날짜 선택), **체크 대상명**(텍스트, 라벨은 선택한 그룹의 subjectLabel 표시. 예: "경로당명"), 체크리스트 그룹 선택
2. **체크 화면**: 선택한 그룹의 항목이 **쭉** 나열
   - 각 항목: 체크박스 + (선택) 항목별 비고
   - 하단 또는 상단에 **전체 비고** 한 줄
3. **저장** 시 → `ChecklistRecord` + `RecordItem[]` 생성 후 로컬 저장 → 기록으로 이동

#### 체크리스트 그룹 관리
- **그룹 목록**: 그룹 추가 / 그룹명 수정 / 그룹 삭제
- **그룹 추가·수정 시**: 그룹명 + **체크 대상명 라벨** 설정 (예: "경로당명", "매장명". 비우면 "대상명")
- **그룹 상세(항목 관리)**:
  - 해당 그룹의 **템플릿 항목** 목록 (드래그로 순서 변경 있으면 좋음)
  - 항목 추가: 제목 입력
  - 항목 수정/삭제

## 3. 로컬 저장
- **react-native-mmkv** 사용 (동기 API, 속도·용량 면에서 AsyncStorage 대비 유리)
- JSON 직렬화 후 MMKV에 저장
- 키 예: `checklist_groups`, `checklist_templates`, `checklist_records`, `record_items`
- 필요 시 한 번에 읽어와서 메모리에서 조인 후 화면에 표시

## 4. 폴더 구조

```
src/
├── components/       # 공용 UI (버튼, 리스트 아이템, 입력 필드 등)
├── constants/        # 스토리지 키, 기본 라벨 등
├── navigation/       # 탭/스택 네비게이션
├── screens/         # 화면 컴포넌트
│   ├── HomeScreen.tsx          # 기록 (홈)
│   ├── RecordDetailScreen.tsx  # 기록 상세
│   ├── NewRecordScreen.tsx      # 새 기록 (입력 → 체크)
│   ├── GroupListScreen.tsx      # 체크리스트 그룹 목록
│   └── GroupDetailScreen.tsx   # 그룹 상세 (템플릿 항목 관리)
├── storage/         # MMKV 래퍼, 읽기/쓰기 로직
├── types/           # ChecklistGroup, ChecklistRecord 등 타입 정의
└── utils/           # ID 생성(uuid), 날짜 포맷 등
```

- **App.tsx**: 네비게이션 감싸기만 하고, 실제 라우트는 `src/navigation`에서 처리.

---

## 5. 플로우 요약
```
[체크리스트 그룹 관리] → 그룹 생성 → 그룹 안에 "항목" 여러 개 추가
        ↓
[새 기록] → 날짜·체크 대상명 입력 → 그룹 선택 → 그룹 항목들이 쭉 나옴 → 하나씩 체크 + 비고 → 저장
        ↓
[기록] → 리스트로 쭉 표시 → 과거 기록 탭해서 상세 보기
```

---

## 6. 추가로 있으면 좋은 것 (선택)
- 기록 **수정**: 이미 저장한 기록의 체크/비고 수정
- 기록 **삭제**
- 그룹/템플릿 **순서** 변경 (드래그 앤 드롭)
- **검색/필터**: 기록에서 날짜 범위, 체크 대상명, 그룹별 필터
- **내보내기**: 기록을 CSV/엑셀 등으로 내보내기 (나중에 백엔드 붙일 때도 유용)

이 스펙대로 구현하면 경로당 방문뿐 아니라 어떤 현장/매장 점검에도 그대로 쓸 수 있습니다.
